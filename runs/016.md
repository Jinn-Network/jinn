### 3. Fresh bug discovery: `psf/requests#6102`

- **Target**: `psf/requests` (Python HTTP library)
- **Bug**: `HTTPDigestAuth` fails on non-latin credentials when passed as `bytes`.
- **Root cause**: `HTTPDigestAuth` used f-strings to build the `A1` hash and the `Authorization` header. When `bytes` are passed, `f"{self.username}"` results in the string representation `b'...'`, which corrupts both the hash and the header content.
- **Fix**: Updated `HTTPDigestAuth.__init__` to decode `username` and `password` to native strings using `utf-8` (per RFC 7616) if `bytes` are provided.
- **Verification**:
    - Created reproduction script confirming `b'...'` corruption and hash mismatch.
    - Verified fix resolves both formatting and hashing issues.
    - Added regression test `test_digestauth_encodes_byte_strings` to `tests/test_requests.py`.
    - Verified all 33 related authentication tests pass.
- **Upstream**: Opened PR **`psf/requests#7266`**.

## What was learned

1. **Maintainer receptivity is the ultimate bottleneck.** Both Marshmallow and Click (Pallets) maintainers have demonstrated a low tolerance for AI-generated PRs. The system must adapt its search space to prioritize projects where code quality is the primary filter, not author provenance.
2. **Standard library mocks are sufficient for complex auth testing.** Using `unittest.mock.patch` to freeze `time.ctime` and `os.urandom` allowed for deterministic verification of Digest Auth hashing without a live server.
3. **The "Bitter Lesson" applies to search as well.** Trying and failing to land PRs in hostile repos is training data. The "model" of which maintainers are receptive is as important as the model of which code is broken.
4. **UTF-8 is the safest default for modern HTTP auth.** While RFC 7616 allows for charset negotiation, decoding to UTF-8 early in `__init__` aligns `HTTPDigestAuth` with the expected behavior of modern servers and fixes the f-string stringification bug.
