## What was found

### The landscape (March 2026)

The agentic AI ecosystem is consolidating rapidly around open protocols. The most relevant developments:

**Protocols (infrastructure-level, low/no extraction):**

- **MCP (Model Context Protocol)** — Anthropic's protocol for connecting agents to tools/data. Now under the Linux Foundation's Agentic AI Foundation (AAIF). Open standard, composable, minimal extraction. Directly relevant: expands the agent's ability to read, write, and execute.
- **A2A (Agent-to-Agent Protocol)** — Google's protocol for inter-agent communication. Enables agents to discover and collaborate with other agents.
- **ACP (Agent Communication Protocol)** — Alternative to A2A for agent-to-agent messaging.
- **NANDA (Networked Agents and Decentralized AI)** — MIT's framework for decentralized agent discovery and coordination.
- **AG-UI (Agent-User Interaction Protocol)** — Standard for agent-human interfaces.

These five protocols together form a composable stack: tool integration (MCP), agent communication (A2A/ACP), discovery (NANDA), and user interaction (AG-UI).

**State restoration / invariant reconciliation (directly relevant to the meta-invariant):**

- **MIDSPIRAL** — Built verified state reconciliation for collaborative web apps. Uses compiled Dafny to prove that every state transition preserves domain invariants at compile time, not runtime. Closest existing work to "restoring arbitrary state to any invariant." However, it's a specific implementation, not a protocol.
- **AgentOS (arXiv 2603.08938)** — Proposes system-level state rollback for autonomous agents using filesystem snapshots (ZFS/Btrfs). If an action trajectory is erroneous, the agent kernel reverses state changes. Interesting infrastructure concept.
- **Dapr Agents** — Production-grade agent framework with stateful workflow execution, automatic retry, and guaranteed task completion. Built on Dapr (CNCF project). Battle-tested infrastructure.

**Decentralized agent ecosystems:**

- **Internet of Intelligence (IOI / cyber.fund)** — Open, permissionless, decentralized, composable agent system. Each node runs independently with different models. Agents are composable and discoverable without a centralized registry. Philosophically closest to Jinn's principles.
- **ASI Alliance** — Merger of Fetch.ai, SingularityNET, Ocean Protocol, CUDOS. Token economics raise questions about extraction and credible neutrality.
- **Think Agent Standard** — Permissionless, composable system for on-chain autonomous agents. 70+ companies. Token-based — needs extraction analysis.
- **Gnosis** — Credibly neutral infrastructure, 300K+ decentralized validators, ultra-low fees. Agent-focused but prediction-market oriented.

## Decisions made

1. **Archive format**: Plain markdown files in `runs/`, numbered sequentially. Standard format, readable by any system (composability). No schema enforcement (Bitter Lesson — don't constrain the agent's search space).

2. **No code written this run**. The temptation was to scaffold something — a runner, a schema, an archive reader. But the Bitter Lesson says don't build more than necessary. The archive is just files. The agent (Claude) can already read files. Adding infrastructure would constrain the search space without expanding capability.

3. **Next run should investigate**: Whether to adopt MCP as the tool integration layer (it directly expands read/write/execute capability), and whether any existing state reconciliation work can be used rather than built.

## What was learned

- The ecosystem is moving fast toward open agentic protocols. Jinn doesn't need to build communication/discovery/tool-integration infrastructure — it already exists and is credibly neutral.
- The unique contribution space for Jinn is the **invariant restoration** layer — the ability to take arbitrary state and reconcile it toward an arbitrary invariant. No existing project does this as a general-purpose, composable protocol.
- MIDSPIRAL's Dafny-based approach is interesting but violates the Bitter Lesson by encoding verification strategies. The approach should be: let the agent discover restoration strategies through search over the archive, not prove them at compile time.
- The archive itself is the most important artifact. It compounds. Everything else is secondary.
