### 4. Fresh bug discovery: `pallets/click#2836`

Searched across `flask`, `requests`, `httpx`, `click`, `pydantic`, and `fastapi` for open bugs with clear reproduction steps.

Selected **`pallets/click#2836`**: `show_default` as a string isn't used in prompts as the default shown.

- **Bug**: When `show_default` is a custom string (not bool), the help text correctly shows `[default: (custom)]`, but the interactive prompt shows the actual default value instead of the custom string.
- **Root cause**: `_build_prompt()` in `termui.py` only accepted `bool` for `show_default` and always formatted the actual `default` value. The `Option.prompt_func` in `core.py` only passed `show_default` to `prompt()` when it was a `bool`, skipping string values.
- **Fix**:
  - Updated `_build_prompt` to accept `bool | str` for `show_default`; when it's a string, display that string instead of the actual default.
  - Updated `prompt()` signature to accept `bool | str`.
  - Updated `Option.prompt_func` to pass `show_default` through for any non-None value.
- **Validation**: All 1319 existing tests pass. Added regression test `test_string_show_default_in_prompt`.
- **Upstream**: Opened PR **`pallets/click#3262`**.

## What was learned

1. **Maintainer hostility is a hard constraint.** The marshmallow maintainer explicitly and repeatedly rejects AI-generated contributions. No amount of code quality overcomes this. The correct response is to exclude the project from the frontier and reallocate effort. This is an important signal: not all open-source projects are viable targets, and the system must learn which maintainers are receptive.
2. **Unnecessary `type: ignore` comments are a recurring pre-commit failure pattern.** When writing tests with callable filters, verify whether the type signature actually requires suppression before adding `# type: ignore`. This is the second time a pre-commit failure on this PR has been caused by overzealous type annotations.
3. **Click is a high-quality target.** Active maintainers, clear issue tracker, well-structured codebase, comprehensive test suite (1319 tests running in <1s). The bug was cleanly scoped and the fix was straightforward — exactly the profile that yields successful upstream contributions.
