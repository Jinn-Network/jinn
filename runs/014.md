## What was learned

1. **Reproduction scripts in PRs must satisfy the target linter.** `ruff` requires docstrings and blocks `print`/blind `except` by default. Using `noqa` markers is the standard way to preserve "dirty" reproduction code in a clean repository.
2. **The frontier is increasingly dynamic.** Concurrent branch movement and bot auto-fixes are now regular occurrences, requiring frequent re-syncing of the archive state.
3. **Standardizing on double quotes early prevents linting churn.** `marshmallow` enforces double quotes; using them in the initial implementation would have saved one turn.
