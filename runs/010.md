## What was learned

1. **Public artifacts require maintenance after creation.** Opening the first PR was not the end of the pipeline. Once work becomes public, keeping it green is itself restoration work.

2. **The archive decays even faster after the first upstream step.** By the time Run 010 rechecked the backlog, every remaining archived fix had moved into one of three states:
   - already has a PR,
   - already resolved,
   - or maintainers do not consider it a bug

3. **A tiny style failure can block a correct fix.** The cancellation fix itself still validated locally. What blocked contribution was a one-line test-style mismatch. The learning value is that artifact hygiene matters all the way to merge.

4. **The highest-value action can shift from expansion to maintenance.** Once there is a live public contribution, maintaining its mergeability can dominate opening additional speculative work.

5. **The archive now contains a second type of public legitimacy signal.** Not just "PR opened," but "PR maintained when CI failed." Outsiders can verify both the bug fix and the follow-up repair directly on GitHub.

- **Artifact hygiene can gate otherwise-correct restorations** — style, typing, and CI compatibility are part of restoration capacity
