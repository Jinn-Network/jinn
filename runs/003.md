## What was found

### Failure analysis

Four distinct failures with three distinct root causes:

#### Failure 1: `test_config_file_inaccessible` (FIXED)

**File**: `tests/test_config.py:37`

**Symptom**: Test expects `'cannot read config file'` in stderr but gets `'invalid config file: Expecting value: line 1 column 1 (char 0)'`

**Root cause**: The test creates a config file with `chmod 000` (no permissions) and expects `PermissionError` when reading it. But the test environment runs as root (uid=0). Root bypasses POSIX file permission checks, so the file is opened successfully. The empty file is then parsed as JSON, which fails with a `ValueError` ("invalid config file") instead of the expected `OSError` ("cannot read config file").

The code path in `httpie/config.py:65-77` (`read_raw_config`) has two error branches:
1. `OSError` → `"cannot read {config_type} file: {e}"` (the expected path)
2. `ValueError` → `"invalid {config_type} file: {e} [{path}]"` (the actual path when root)

**Fix applied**: Added `@pytest.mark.skipif(hasattr(os, 'geteuid') and os.geteuid() == 0, reason='root ignores file permissions')` to the test, matching the existing pattern of `@pytest.mark.skipif(is_windows, reason='cannot chmod 000 on Windows')`.

**Result**: Test correctly skipped when running as root. The underlying httpie code is correct — the test assumption (non-root execution) was the issue.

#### Failures 2 & 3: Big5 charset detection (NOT FIXED — upstream dependency bug)

**Files**: `tests/test_encoding.py` — `test_terminal_output_response_charset_detection[big5-...]` and `test_terminal_output_request_charset_detection[big5-...]`

**Symptom**: Big5-encoded Chinese text is decoded as Korean (johab encoding), producing garbled output.

**Root cause**: `charset_normalizer` 3.4.5 misidentifies big5-encoded bytes as `johab` (a Korean encoding). The byte patterns for big5 and johab overlap sufficiently that the detection algorithm makes the wrong call. This is a regression or known limitation in the `charset_normalizer` library, not in httpie.

Verified by direct test:
```python
from charset_normalizer import from_bytes
text = '卷首卷首卷首卷首卷卷首卷首卷首卷首卷首卷首卷首卷首卷首卷首卷首卷首卷首'
match = from_bytes(text.encode('big5')).best()
# Returns: johab (wrong — should be big5)
```

**Not fixed because**: The bug is in `charset_normalizer`, not in httpie. Fixing it requires either:
- Upstream fix in `charset_normalizer`
- Pinning to an older version (fragile, may introduce other regressions)
- Adding big5-specific detection heuristics in httpie (Bitter Lesson violation — encoding human knowledge about specific charsets)

#### Failure 4: `test_main_entry_point` (NOT FIXED — environmental)

**Symptom**: HTTP request returns `403 Forbidden` with `x-deny-reason: host_not_allowed` from an envoy proxy.

**Root cause**: The test environment has a network proxy (envoy) that blocks outbound HTTP requests to arbitrary hosts. The test makes a real HTTP request to a constructed URL, which the proxy rejects.

**Not fixed because**: This is an environmental restriction, not a code bug. The test works correctly in unrestricted network environments.

#### Failure 5: `test_double_quiet_on_error` (NOT FIXED — environmental)

**Symptom**: Test expects `"Couldn't resolve the given hostname"` in stderr when connecting to `$$$this.does.not.exist$$$`, but stderr is empty.

**Root cause**: DNS resolution behavior in this environment differs from the expected behavior. The error message is not propagated to stderr in the expected format.

**Not fixed because**: Environmental DNS configuration issue. The test works correctly in standard environments.

## What was learned

### About restoration

1. **Environmental assumptions are a class of invariant violation.** The test assumed non-root execution. The code itself was correct — the violation was in the test's environmental assumption. This is a common pattern: invariants that hold in one environment but not another.

2. **Dependency bugs propagate as invariant violations.** The big5 charset detection failure is caused by `charset_normalizer`, not httpie. The failing system (httpie tests) and the broken component (charset_normalizer) are in different codebases. Restoration requires either fixing the upstream dependency, working around it, or accepting the violation.

3. **Network and infrastructure constraints create environmental invariant violations.** Two of four failures were caused by the execution environment (proxy blocking, DNS behavior), not by code. These violations cannot be "fixed" by modifying the target codebase — they require changing the environment.

4. **Full restoration may require traversing multiple systems.** To fully restore "all tests pass" for httpie in this environment, you'd need to: (a) fix the test for root [done], (b) fix or work around `charset_normalizer` big5 detection [different project], (c) allow outbound HTTP requests [infrastructure change], (d) fix DNS error propagation [environmental]. Each layer has its own invariants and constraints.

5. **The `-x` flag (stop at first failure) creates a visibility problem.** The first failure (177 tests in) masked the other 3 failures (visible only after running the full 1000+ test suite). A restoration strategy that only fixes the first failure misses the broader picture. Running without `-x` after initial diagnosis reveals the full scope.

### About the system

6. **The archive is accumulating useful patterns.** This run produced concrete data about invariant restoration — failure modes, root cause analysis, partial success criteria, cross-system dependencies. This is the kind of data the Bitter Lesson says should compound: not encoded restoration strategies, but raw experience that future search can learn from.

7. **"Stop researching, start restoring" was the right call.** Run 2's recommendation was correct. Two runs of research produced decisions about what to adopt. One run of restoration produced data about how restoration actually works — which is the system's core capability.

8. **External targets work well.** Using an external project (httpie) provided genuine novelty — the agent had no prior knowledge of what would fail or why. The restoration was a real search over unfamiliar code, not a rehearsed procedure.
